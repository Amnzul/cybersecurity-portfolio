Certificate SSH Login Project Overview

I am assuming SSH Server is already running on your target server

To use an SSH certificate (key-based authentication) instead of a password to log in remotely we need to first generate a key 
ssh-keygen -t rsa -b 2048
  1. ssh-keygen is the command-line utility used to create authentication keys
  2. -t rsa is used to specify which algorithm is used to generate the keys. In this case the RSA (Rivest-Shamir-Adleman) algorithm, which is one of the most widely used for SSH authentication.
  3. -b 2048 specifies how many bits the key is. This key will be 2048 bits long. 4096 bit keys can be used for enhanced security, but they may result in slower performance.
  4. The output will include two files: the private key (which should be kept secure) and the public key (which can be shared with servers to allow secure access).
  5. The default file locations for the private key (id_rsa) and public key (id_rsa.pub) is in the ~/.ssh directory.

This command also works on Microsoft Windows.

Copy Your Public Key to the Linux Server:

On Linux:

Use the ssh-copy-id command to install your public key on the remote server:

ssh-copy-id user@remote_server_ip

On Windows:

Microsoft users will do not have the ssh-copy-id command and will have copy their key using another method, such as to download WinSCP.

Verify SSH Certificate Key Based Login is Working:

Attempt to SSH into the server:

ssh user@remote_server_ip

If set up correctly, you should log in without being asked for a password.

Conclusions

In the course of this project I learned about various algorithms used to generate keys, and how bit length affects security. Apparently there are still some old systems in the wild with insecure key lengths that are vulnerable and appear in the news from time to time after a breach.

Projects for the future might include trying to break a 512 bit key, or setting up a CA (Certificate Authority)
